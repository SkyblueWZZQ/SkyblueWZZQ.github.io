工程化开发工具-Gulp

发表于 2017-03-15 | 分类于 前端学习-工程化工具

　　工程化是前端开发工作中比较重要的一环，开发的项目文件众多，体积庞大，所以需要对文件进行打包、压缩、校验等工作，而人力去处理这些事情是很有风险的，而且效率低下，因此，前端工程化工具就出现了，今天就学习一款前端工程化工具—-Gulp。

安装

  • 安装Node.js
    首先，最基本也最重要的是，我们需要搭建node环境。访问 Nodejs官网 ，然后点击左边的LTS版本（稳定版）绿色按钮下载，下载完成后直接运行程序，一路点击next，直到安装完成。
    检查node是否安装成功：
    　　打开windows系统的命令提示符输入node -v回车，会显示node版本。
  • 安装Gulp
    打开windows系统的命令提示符输入npm install -g gulp回车，等待gulp安装完成。
  • Gulp模块本地化
    Gulp使用的时候要将模块复制到本地（项目）文件夹中，使用windows命令行定位到当前项
    目文件根目录，然后输入npm link gulp，完成之后会在本地文件夹中出现一个“node_modules”文件夹，里面包含的是gulp的文件。

创建入口文件

　　在一个项目中，需要创建一个gulpfile.js文件，这是Gulp的入口文件，我们可以在这个文件中定义一些配置。Gulp通过执行这个文件，会根据我们配置的一些功能信息去实现我们想要的功能。

API

　Gulp一共只有5个API：task、src、pipe、dest、watch、（run）

task

功能：创建任务
说明：我们对文件要进行压缩、打包等操作就是任务，通过task这个API就可以创建任务了。
语法：gulp.task(name,[deps], fn)
　　　name：任务名称，自定义的，是一个字符串
　　　deps：任务依赖集合，是当前任务所依赖的任务集合，依赖集合中的任务会先于当前任务执行。
　　　fn：回调函数，该函数定义任务所要执行的一些操作，通常来说它是这种形式：gulp.src().pipe(someplugin())

通常有两种传参方式 1、gulp.task(name,fn)　2、gulp.task(name,[deps])

// 创建任务
gulp.task("taskname",function(){
    // 一些处理语句
})

src

功能：读取文件，将文件转换成流形式
说明：我们在任务中需要对文件进行操作，所以要读取文件，然后进行操作，src就是读取文件的
语法：gulp.src(path/globs)
　　　path/globs：参数可以是文件的真是路径+文件名+后缀，也可以是globs语法的文件路径+后缀

globs语法是专门为路径设计的一套匹配规则，具体内容请自行科普 0.0

  • 真实路径
// 读取文件
gulp.src('js/demo.js')
  • globs
// 读取文件
gulp.src('js/**.js')

pipe

功能：对gulp.src() 转化的流进行一些处理
说明： 通常在gulp.src()后面使用
语法：.pipe(methods)
　　　参数就是要处理的事情（处理文件的相关方法），例如：压缩、打包、重命名等 

// 读取文件
gulp.src('demo/js/*.js')
     // 进行压缩
     .pipe(uglifyJS())

dest

功能：写入文件
说明：能被pipe进来，并会将处理后的流写入到文件夹，如果指定文件夹不存在，它会创建一个。
语法：gulp.dest(path)
　　　path：要写入文件的路径

// 读取文件
gulp.src('js/demo.js')
    // 输出
    .pipe(gulp.dest('js/myFolder'));

watch

功能：监视文件变动
说明：当监视的文件发生变动时，可以做一些事情
语法：1、gulp.watch(path/globs,[deps])
　　　path/globs：监听的文件路径+文件名+后缀
　　　deps：文件变动后要执行的任务集合
　　　2、gulp.watch(path/globs,fn)
　　　path/globs：监听的文件路径+文件名+后缀
　　　fn：回调函数，文件变动时要执行的事情

// 监听文件变动
gulp.watch('demo/js/*.js',['rename'])

// 监听文件变动
gulp.watch('demo/js/*.js',function(){
    // 一些处理语句
})

run

功能：执行创建的任务
说明：在gulp的新版本中，建议我们用default任务代替run方法，当gulp检测到default任务的时候，就会立即执行defalut依赖的任务
语法：gulp.run(name)　使用default任务：gulp.task('default',[deps])

// 执行创建的任务
gulp.run('showMsg')

// 创建default任务，执行依赖任务
gulp.task('default', ['showMsg']);

应用

// 1、使用gulp，需要引入模块
var gulp = require("gulp");
// 2、创建任务
gulp.task('monitor',function(){
    // 读取文件
    gulp.src('js/demo.js')
        // 写入文件
        .pipe(gulp.dest('dest'));
});
// 监听事件
gulp.watch('js/demo.js',['monitor']);

插件

Gulp本身体积很小，其强大的功能是通过众多插件实现的。

下载Gulp插件

打开windows命令提示符，输入npm install -g gulp-rename

使用插件

在gulpfile.js文件中引入该插件：var rename = require("gulp-rename")，然后在pipe管道中应用插件.pipe(rename('newname'))
完整的程序：

// 引入模块
var gulp = require("gulp");
// 引入插件
var rename = require('gulp-rename');
// 创建任务
gulp.task('renameFile', function() {
    // 指定要操作的文件
   gulp.src('js/demo.js')
    // 新名称
   .pipe(rename('newname.js'))
    // 保存文件的路径
   .pipe(gulp.dest('dest'));
});
// 定义默认任务
gulp.task('default', ['renameFile']);

　　插件的使用方法都一样，都是在pipe中应用，只是有的插件需要传递参数，有的不需要，还是要根据具体的插件而异。

完
